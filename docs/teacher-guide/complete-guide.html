Step 1 — Room 1: The Corrupted Login Banner (~7 min)

Syllabus: §3.1 Number systems · §3.2 Character sets · Two's complement

What students see

A corrupted login banner with five data blocks:

BlockValueType
BLOCK_A010100118-bit binary
BLOCK_B111111118-bit binary (two's complement)
BLOCK_C1000000009-bit binary (invalid)
BLOCK_D010001018-bit binary
CHECKSUM85Denary (distractor)

Step-by-step solution

Question 1: BLOCK_A (01010011) as unsigned denary

Convert 8-bit binary to denary by summing place values:

Bit positions:  128  64  32  16   8   4   2   1
BLOCK_A:          0   1   0   1   0   0   1   1

64 + 16 + 2 + 1 = 83
Answer: 83 (This is ASCII character 'S', but the fragment comes from a separate question.)

Question 2: Denary 69 as hexadecimal (2 digits)

Divide by 16 repeatedly:

69 ÷ 16 = 4 remainder 5
Therefore 69₁₀ = 45₁₆

Check: BLOCK_D is 01000101 = 64+4+1 = 69 = ASCII 'E'.

Answer: 45

Question 3: BLOCK_B (11111111) in 8-bit two's complement

In 8-bit two's complement, the MSB indicates sign. All bits set in 8 bits represents −1.

11111111 (8-bit two's complement) = -1

This is a distractor — not a printable ASCII banner character.

Answer: -1

Question 4: Which block is invalid due to bit-length?

ASCII bytes are exactly 8 bits. BLOCK_C has 9 bits (100000000) — overflow/invalid.

Answer: C

Question 5: Key fragment (2-digit hex)

Enter the hex from Question 2.

Answer: 45 · Fragment E4 · Data Link layer (2)
Common errors: Treating 11111111 as denary 255; decoding the 9-bit block; using checksum 85 → hex 55 (that is 'U', a distractor).

Step 2 — Room 2: The Network Trace (~7 min)

Syllabus: §4.1 LAN/WAN · DNS · TCP/IP · packet switching · public/private IP

What students see

A network map (client 192.168.12.8 → gateway/NAT → server 203.0.113.44:443) and five shuffled packet events to reorder, plus a security implication multiple-choice question.

Step-by-step solution

Part A — Order events (earliest first)

  1. DNS lookup — Hostname archive.uni-lab.edu must resolve to IP 203.0.113.44 before any connection.
  2. TCP SYN — Client initiates TCP handshake to port 443.
  3. Router forward — Packet NAT'd (192.168.12.8 → public) and routed via ISP.
  4. HTTP GET /vault — TLS established; request/response over encrypted channel.
  5. TCP FIN — Connection closed after response.
Order: DNS → SYN → Route → HTTP → FIN

Part B — Security implication

The trace shows private IP 192.168.12.8 — a reconnaissance risk if logs are exposed (attackers map internal topology). TLS is in use (not HTTP-only); DNS being down would prevent connection entirely.

Answer: Private IP visible in trace — reconnaissance risk

No separate fragment field in Room 2 — completing the room awards fragment automatically on success.

Fragment: 43 (ASCII C) · Fragment C3 · Network layer (3)

Step 5 — Room 5: The Logic Lock (~6 min)

Syllabus: §3.2 Logic gates · §15.2 Boolean algebra

English rule

Open only if the admin card is valid AND either the biometric match is true OR the emergency override is active — but never if lockdown is active.

Variables: A = admin card · B = biometric · E = emergency override · L = lockdown

Step 6 — Room 6: The Algorithm Vault (~7 min)

Syllabus: §9.2 Searching/sorting · §19.1 Recursion · ADTs · Big O

Step-by-step solution

1. Search type (list unsorted after breach)

Binary search requires sorted data. The pseudocode falls through to the FOR loop (linear search) when unsorted.

Answer: linear search

2. ADT for recursive call frames

Each recursive call pushes a stack frame; returns pop frames — LIFO = stack.

Answer: Stack (LIFO)

3. Trace factorial(4)

factorial(4) = 4 × factorial(3)
factorial(3) = 3 × factorial(2)
factorial(2) = 2 × factorial(1)
factorial(1) = 1

Unwind: 1 → 2×1=2 → 3×2=6 → 4×6=24
Answer: 24

4. Merge sort complexity

Merge sort is O(n log n) — not O(n²) like bubble sort.

Answer: O(n log n)

5. Fragment field

E = denary 69 = hex 45.

Fragment: 45 · Fragment E9 · Application layer (7)

Answer: OOP (paradigm mixing procedures with objects)

Appendix A — Full hint sequences

Room 1

  1. Each valid character byte is exactly 8 bits. Group binary in eights before converting to denary.
  2. 01010011 → 83 denary. To get the fragment, convert denary 69 to hexadecimal.
  3. 69 ÷ 16 = 4 remainder 5, so the fragment is 45. Ignore 11111111 (−1) and the 9-bit overflow value.

Room 2

  1. DNS must resolve the hostname before any TCP connection to the server IP.
  2. Order: DNS → SYN → routed request → HTTP 200 → FIN. Pick the reconnaissance risk about private IPs.
  3. Event order: dns, syn, route, http, fin. Security: private IP leak. Fragment: 43.

Room 3

  1. Ask: does the countermeasure match the attack vector? Strong passwords do not stop phishing.
  2. Phishing→training; Pharming→certs; Ransomware→backup; Spyware→anti-spyware; Brute→lockout; Hospital→integrity.
  3. Full match table in main guide. Fragment: 52.

Room 4

  1. Bulk data uses symmetric encryption. Key exchange and signatures use asymmetric keys.
  2. Encrypt with recipient public key; decrypt with private. Sign with sender private; verify with public.
  3. Reject UnknownCA certificate. Fragment: 55.

Room 5

  1. Lockdown active (L=1) should force the door closed regardless of other sensors.
  2. Structure: admin AND (biometric OR emergency) AND NOT lockdown.
  3. Expression: A AND (B OR E) AND NOT L. Truth rows: (1,0,0,0)→1; (1,0,0,1)→0; (0,1,1,0)→0. Fragment: 53.

Room 6

  1. Binary search needs sorted data. Recursion uses a call stack — last in, first out.
  2. Unsorted → linear search. Stack ADT. factorial(4)=24. Merge sort → O(n log n).
  3. Answers: linear, stack, 24, O(n log n). Fragment: 45.

Vault

  1. Each fragment card shows its stack layer. Sort from layer 1 upward.
  2. Order: S1(53) → E4(45) → C3(43) → U7(55) → R5(52) → E9(45).
  3. Concatenated hex converts to ASCII unlock phrase: SECURE.
