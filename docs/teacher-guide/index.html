Room 2 — The Network Trace

Syllabus §4.1 · DNS, TCP/IP, routing · ~7 min · Fragment: 43

Event order: DNS lookup → TCP SYN → Router forward → HTTP GET → TCP FIN
Security: Private IP visible in trace — reconnaissance risk
Fragment: 43 (ASCII C)
Misconception: Placing HTTP before the TCP handshake, or DNS after connection setup.

Room 3 — The Threat Console

Syllabus §6.1 · Threats & countermeasures · ~7 min · Fragment: 52

IncidentCountermeasure

Pharming / DNS redirectVerify digital certificates & monitor DNS/hosts
RansomwareOffline backups + anti-malware scan
Spyware / keyloggerAnti-spyware + least-privilege accounts
Brute-force loginAccount lockout + two-factor authentication
Altered audit recordsIntegrity checks (hashing/audit trails)
Fragment: 52 (ASCII R)
Key teaching point: A strong password does not stop phishing. Altered records = integrity, not just privacy.

Plenary questions

  1. How does each puzzle room map to a layer of the TCP/IP model?
  2. Why is phishing not defeated by strong passwords alone?
  3. What is the difference between data privacy and data integrity?
  4. Why does TLS use both symmetric and asymmetric encryption?
  5. When can binary search be used — and when must you use linear search?
  6. Which single countermeasure would not alone have stopped this fictional breach?
